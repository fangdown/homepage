# 个人主页 - 需求文档

## 1. 项目概述

- **项目名称**: 个人主页 / Fangdu
- **项目类型**: 静态个人网站
- **核心功能**: 展示个人作品集，作为个人品牌入口
- **目标用户**: 访客、潜在雇主/客户、技术同行

## 2. UI/UX 设计规范

### 2.1 整体风格
- **设计风格**: 赛博朋克 + 极简现代（参考 aiking.dev/works）
- **主题**: 深色主题
- **配色方案**:
  - 背景色: `#0a0a0a` (近黑色)
  - 卡片背景: `#111111` (深灰)
  - 主文字: `#ffffff` (白色)
  - 次要文字: `#888888` (灰色)
  - 强调色/金色: `#D4AF37` (AI-gold)
  - 边框色: `#333333`

### 2.2 布局结构
```
┌─────────────────────────────────────────────┐
│  Navigation (导航栏)                         │
│  Logo/Name          Works  Blog  About      │
├─────────────────────────────────────────────┤
│                                             │
│  Hero Section (个人介绍区)                   │
│  姓名 + 职位 + 一句话介绍                     │
│                                             │
├─────────────────────────────────────────────┤
│                                             │
│  Works Section (作品集区域) - 当前唯一功能    │
│  ┌─────┐  ┌─────┐  ┌─────┐                 │
│  │Card │  │Card │  │Card │                 │
│  └─────┘  └─────┘  └─────┘                 │
│                                             │
├─────────────────────────────────────────────┤
│  Footer (页脚)                              │
│  © 2026 社交链接                            │
└─────────────────────────────────────────────┘
```

### 2.3 组件设计

#### 导航栏 (Navbar)
- 固定顶部，深色半透明背景
- 左侧: Logo/姓名
- 右侧: Navigation links (Works, Blog, About)
- 滚动时添加底部边框

#### Hero Section
- 大标题 (姓名)
- 副标题 (职位/角色)
- 简短描述 (一句话介绍)
- 可选: 社交图标链接

#### 作品卡片 (Work Card)
- 尺寸: 响应式，3列网格 (桌面) / 2列 (平板) / 1列 (手机)
- 内容:
  - 项目名称
  - 项目描述 (1-2行)
  - 技术标签 (Skills tags)
  - GitHub 链接图标
  - 可选: 外部链接图标
- 交互:
  - Hover: 边框变为金色，轻微上浮 (translateY -4px)
  - 过渡动画: 300ms ease

#### 页脚 (Footer)
- 居中显示版权信息
- 社交链接图标 (GitHub, Twitter/X, Email 等)

## 3. 功能需求

### 3.1 作品集列表 (当前阶段)
- 静态展示作品数据
- 支持分类/筛选（可选，后期）
- 展示字段: 标题、描述、技术栈、链接

### 3.2 响应式设计
- Desktop: ≥1024px (3列网格)
- Tablet: 768px - 1023px (2列网格)
- Mobile: <768px (1列网格)

### 3.3 动画效果
- 页面加载: 简单的 fade-in 效果
- 卡片 hover: 边框高亮 + 上浮
- 平滑滚动

## 4. 技术栈

- **框架**: Next.js 14+ (App Router)
- **样式**: Tailwind CSS
- **字体**: 系统字体栈 + Google Fonts (可选)
- **图标**: Lucide React 或 Heroicons
- **部署**: Vercel (预留)

## 6. 数据结构

### 作品数据模型
```typescript
interface Work {
  id: string;
  title: string;
  description: string;
  tags: string[];
  githubUrl?: string;
  externalUrl?: string;
  featured?: boolean;
}
```

## 7. 验收标准

1. 页面在桌面端和平板端显示正常
2. 作品卡片 hover 效果正常
3. 导航链接可点击（跳转锚点或占位页面）
4. 无 console error
5. 加载速度 < 2s (生产环境)
